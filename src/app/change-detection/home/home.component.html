Change detection with OnPush:

  @Component({{'{'}}
    changeDetection: ChangeDetectionStrategy.OnPush
    ...
  )

If you use OnPush in a child component, when you make changes to a binded property from parent component, the child component is not notified, only is notified if you change the whole object/property.

In the next example, the "Toggle First Item" button calls a function that changes the "completed" property of the first TODO object in TODOS collection, on the other hand the "Add Todo to List" button calls a function that makes a copy of the original collection, push a new Todo inside, and sets the collection, this way the child component is notified.

If you remove the OnPush, on both cases the child component is notified, give it a check!
